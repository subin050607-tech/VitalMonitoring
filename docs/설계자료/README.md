## 구현과의 대응

- **요구사항서**의 기능 요구사항(FR-01 실시간 관제 / FR-02 위험 기준치 / FR-03 바이탈 추세 / FR-04 병동 통계)이
  루트 [README](../../README.md)의 화면 표와 위험 기준치 항목으로 구현됨.
- **운영 시나리오**의 데이터 설계(개체·관계·ERD·쿼리)는 [`supabase/recover_schema.sql`](../../supabase/recover_schema.sql)의
  테이블·뷰·외래키(FK) 정의와 일치. 개체 간 1:N 관계는 실제 DB에 FK 제약으로 적용되어 있음.
- **상세설계서**의 화면 설계는 `components/` 하위 화면별 컴포넌트로 구현됨.
